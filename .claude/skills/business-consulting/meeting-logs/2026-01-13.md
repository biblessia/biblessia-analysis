## 핵심 결정: "교회 이야기" 기능으로 수익화

### 왜 이 방향인가?

**문제 인식**
- 무료로만 확장하면 나중에 유료화 어려움
- 교회가 돈을 낼 명확한 이유가 필요

**목회자 인사이트**
> "목회자들이 피드백을 받지 못해 외롭다"

### "교회 이야기"란?

**컨셉**: 교회 내 활동을 성도들과 공유하는 소통 창구

| 이것은 | 이것이 아님 |
|--------|------------|
| 이미 앱에 있는 성도와 소통 | 모든 성도가 가입해야 함 |
| 이벤트 있을 때 공유 | 매주 올려야 하는 의무 |
| 스토리 공유 | 공지사항 |

**사용 예시**
- 주일학교 부장님: "오늘 아이들이랑 성경 퀴즈했어요"
- 전도팀: "길거리 전도 다녀왔습니다"
- 청년부: "양로원 봉사 다녀왔습니다"
- 목사님: "이번 주 설교 준비하며 받은 은혜"

**작성자**: 목사님, 전도사님, 각 부서 부장님

## 교회 접근 전략: 홈페이지 + 비블레시아

### 현재 상황
- 3개 교회 중 2개 교회 홈페이지 제작 중
- AI 활용으로 빠르게 홈페이지 제작 가능

### 시너지 구조

```
홈페이지 제작 (교회 접근 채널)
        ↓
비블레시아 도입 제안
        ↓
교회 이야기 축적
        ↓
홈페이지 "은혜의 기록" 페이지 자동 생성
```

### 교회가 얻는 추가 가치

| 기존 홈페이지 | 비블레시아 연동 시 |
|--------------|-------------------|
| 정적인 소개 페이지 | 살아있는 교회 히스토리 |
| 직접 업데이트 필요 | 앱 데이터로 자동 업데이트 |

**예시: "2026년 우리 교회 이야기" 페이지**
- 1월: 청년부 양로원 봉사
- 2월: 길거리 전도 - 3분 복음 전함
- 3월: 주일학교 부활절 행사
- ...

→ 연말에 "올해 하나님께서 우리 교회를 인도해주신 은혜" 자동 정리

### 영업 포인트

> "홈페이지 만들어드리고요, 비블레시아 앱도 연동해드릴게요.
> 앱에서 교회 이야기 올리시면 홈페이지에 자동으로 반영돼요.
> 연말에 '올해 우리 교회 은혜의 기록' 페이지가 자동으로 만들어져요."

## 다음 스텝: 목회자 인터뷰

### 인터뷰 설명 스크립트

> "지금 비블레시아에 우리 교회 성도분들 100명 정도 계시잖아요.
> 이분들한테 교회에서 일어나는 일들을 공유할 수 있는 '교회 이야기' 공간을 만들려고 해요.
>
> 주일학교 부장님이 아이들 활동 올리고, 전도팀이 전도 나간 이야기 올리고,
> 청년들이 봉사 다녀온 거 올리면 앱 쓰는 성도분들께 알림이 가고
> 좋아요나 댓글로 반응할 수 있어요."

### 핵심 질문

1. **니즈 확인**
   - "이런 공간 있으면 좋으실 것 같으세요?"
   - "우리 교회에서 어떤 분들이 글을 올리실 것 같으세요?"
   - "어떤 내용들이 주로 올라올 것 같으세요?"

2. **과금 방향 언급 (구체적 가격 X)**
   - "나중에 성도 수에 따라 과금하는 방식으로 생각하고 있어요"
   - "일단 먼저 써보시겠어요?"

3. **확산 가능성**
   - "다른 교회 목사님들께 소개해주실 의향 있으세요?"

### 인터뷰에서 확인해야 할 것

| 확인 항목 | 왜 중요한가 |
|-----------|------------|
| 실제로 쓸 의향이 있는가 | "좋은 아이디어"와 "실제 사용"은 다름 |
| 누가 글을 올릴 것인가 | 목사님만? 부장님들도? → MVP 범위 결정 |
| 어떤 내용을 올릴 것인가 | 실제 사용 시나리오 구체화 |
| 과금 방향에 거부감 없는가 | 나중에 유료화 가능성 확인 |
| 다른 교회에 소개 가능한가 | 확장 가능성 |

### 인터뷰 → 테스트 → 가격 협의 순서

```
인터뷰: 니즈 확인 + "나중에 과금" 언급 + 테스트 제안
    ↓
테스트: 2-4주 사용 → 가치 체감
    ↓
가격 협의: 실제 사용 후 구체적 가격 논의
```

**가격은 가치를 느낀 후에 이야기한다**
